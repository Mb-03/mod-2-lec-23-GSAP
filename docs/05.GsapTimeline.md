# GSAP Timeline - Complete Guide

## üéØ What is a Timeline?

A **Timeline** is like a **movie director** for your animations. Instead of animations starting all at once, a timeline lets you:

- **Chain animations** one after another
- **Overlap animations** precisely
- **Control multiple animations** as one group
- **Pause, play, reverse, or restart** all animations together

### üé¨ Real-World Analogy

Think of it like a **movie scene**:

- Without timeline: All actors start moving at the same time (chaos!)
- With timeline: Actor 1 moves ‚Üí Actor 2 enters ‚Üí Both dance together ‚Üí Actor 3 appears

---

## üìö THEORY: Why Use Timelines?

### ‚ùå WITHOUT Timeline (Messy):

```javascript
gsap.to(".box1", { x: 200, duration: 1 });
gsap.to(".box2", { x: 200, duration: 1, delay: 1 }); // Manual delay
gsap.to(".box3", { x: 200, duration: 1, delay: 2 }); // Manual delay
```

**Problems:**

- Hard to control timing
- Can't pause/reverse all together
- Manual delay calculations
- Difficult to modify

### ‚úÖ WITH Timeline (Clean):

```javascript
const tl = gsap.timeline();
tl.to(".box1", { x: 200, duration: 1 })
  .to(".box2", { x: 200, duration: 1 })
  .to(".box3", { x: 200, duration: 1 });
```

**Benefits:**

- Automatic sequencing
- One control for all (pause, play, reverse)
- Easy to modify timing
- Clean, readable code

## üèóÔ∏è Timeline Basics

### Creating a Timeline

```javascript
const tl = gsap.timeline();

// With options
const tl = gsap.timeline({
  repeat: 2, // Repeat entire sequence 2 times
  repeatDelay: 1, // Wait 1 second between repeats
  yoyo: true, // Play forward then backward
  paused: true, // Start paused (play later with tl.play())
  defaults: {
    // Default settings for ALL animations
    duration: 1,
    ease: "power2.inOut",
  },
});
```

## ‚è±Ô∏è POSITION PARAMETER - The Secret Sauce!

The **position parameter** controls WHEN an animation starts in the timeline.

### 1Ô∏è‚É£ **No Position** (default) - One After Another

```javascript
tl.to(".box1", { x: 200 })
  .to(".box2", { x: 200 }) // Starts AFTER box1 finishes
  .to(".box3", { x: 200 }); // Starts AFTER box2 finishes
```

### 2Ô∏è‚É£ **Absolute Time** - Start at Exact Second

```javascript
tl.to(".box1", { x: 200 })
  .to(".box2", { x: 200 }, 2) // Start at 2 seconds
  .to(".box3", { x: 200 }, 5); // Start at 5 seconds
```

### 3Ô∏è‚É£ **Relative Time "+=1"** - Wait Extra Time

```javascript
tl.to(".box1", { x: 200 })
  .to(".box2", { x: 200 }, "+=1") // Wait 1 second AFTER box1
  .to(".box3", { x: 200 }, "+=0.5"); // Wait 0.5 seconds AFTER box2
```

### 4Ô∏è‚É£ **Relative Time "-=1"** - Overlap by Time

```javascript
tl.to(".box1", { x: 200, duration: 6 })
  .to(".box2", { x: 200 }, "-=2") // Start 1 second BEFORE box1 ends
  .to(".box3", { x: 200 }, "-=0.5"); // Start 0.5 seconds BEFORE box2 ends
```

### 5Ô∏è‚É£ **"<"** - Start with Previous Animation

```javascript
tl.to(".box1", { x: 200 })
  .to(".box2", { y: 200 }, "<") // Start at SAME TIME as box1
  .to(".box3", { rotation: 360 }, "<"); // Also start with box1
```

### 6Ô∏è‚É£ **">"** - Start at End of Previous

```javascript
tl.to(".box1", { x: 200 })
  .to(".box2", { x: 200 }, ">") // Same as default (start after)
  .to(".box3", { x: 200 }, ">1"); // Start 1 second after previous ends
```

### 7Ô∏è‚É£ **Labels** - Named Checkpoints

```javascript
tl.to(".box1", { x: 200 })
  .addLabel("midpoint") // Create a label
  .to(".box2", { x: 200 })
  .to(".box3", { x: 200 }, "midpoint") // Start at the label
  .to(".box4", { x: 200 }, "midpoint+=1"); // Start 1s after label
```

## üéÆ Timeline Control Methods

```javascript
const tl = gsap.timeline({ paused: true });

// Playback control
tl.play(); // Start playing
tl.pause(); // Pause
tl.resume(); // Resume from pause
tl.reverse(); // Play backwards
tl.restart(); // Start from beginning

// Speed control
tl.timeScale(2); // Play 2x speed
tl.timeScale(0.5); // Play 0.5x speed (slow motion)

// Jump to time
tl.seek(2); // Jump to 2 seconds
tl.progress(0.5); // Jump to 50% complete

// Info
tl.duration(); // Get total duration
tl.time(); // Get current time
```

## üéì Advanced Timeline Techniques

### 1. **Nested Timelines**

```javascript
const masterTL = gsap.timeline();
const scene1 = gsap.timeline();
const scene2 = gsap.timeline();

scene1.to(".actor1", { x: 100 }).to(".actor2", { y: 100 });
scene2.to(".prop1", { rotation: 360 }).to(".prop2", { scale: 2 });

masterTL.add(scene1).add(scene2); // Add entire timelines
```

### 2. **Timeline with Callbacks**

```javascript
const tl = gsap.timeline({
  onStart: () => console.log("Timeline started!"),
  onComplete: () => console.log("All done!"),
  onUpdate: () => console.log("Progress:", tl.progress()),
  onRepeat: () => console.log("Repeating..."),
});
```

### 3. **Dynamic Timeline Creation**

```javascript
const tl = gsap.timeline();

// Add animations dynamically
boxes.forEach((box, i) => {
  tl.to(
    box,
    {
      x: 200,
      duration: 1,
      backgroundColor: `hsl(${i * 60}, 70%, 50%)`,
    },
    i * 0.2
  ); // Stagger by 0.2 seconds
});
```

## üéØ Common Patterns

### Pattern 1: **Stagger Effect**

```javascript
const tl = gsap.timeline();
tl.to(".box", {
  x: 200,
  duration: 1,
  stagger: 0.2, // Each box starts 0.2s after the previous
});
```

### Pattern 2: **Wave Effect**

```javascript
const tl = gsap.timeline({ repeat: -1 });
tl.to(".box1", { y: -50, duration: 0.5 })
  .to(".box2", { y: -50, duration: 0.5 }, "-=0.3")
  .to(".box3", { y: -50, duration: 0.5 }, "-=0.3")
  .to(".box1", { y: 0, duration: 0.5 }, "-=0.3")
  .to(".box2", { y: 0, duration: 0.5 }, "-=0.3")
  .to(".box3", { y: 0, duration: 0.5 }, "-=0.3");
```

### Pattern 3: **Loading Sequence**

```javascript
const tl = gsap.timeline();
tl.from(".logo", { opacity: 0, scale: 0, duration: 1, ease: "back.out" })
  .from(".title", { opacity: 0, y: -50, duration: 0.5 }, "-=0.3")
  .from(".content", { opacity: 0, y: 50, duration: 0.5 }, "-=0.2")
  .from(".button", {
    opacity: 0,
    scale: 0,
    duration: 0.3,
    ease: "elastic.out",
  });
```

## üí° Pro Tips

1. **Use `defaults`** for repeated properties:

```javascript
const tl = gsap.timeline({
  defaults: { duration: 1, ease: "power2.inOut" },
});
tl.to(".box1", { x: 200 }) // Uses defaults
  .to(".box2", { x: 200 }); // Uses defaults
```

2. **Store timeline in ref** for React:

```javascript
const tl = useRef();

useGSAP(
  () => {
    tl.current = gsap.timeline();
    tl.current.to(".box", { x: 200 });
  },
  { scope: container }
);

// Later use: tl.current.play()
```

3. **Use `addPause()`** for interactive sequences:

```javascript
const tl = gsap.timeline();
tl.to(".box1", { x: 200 })
  .addPause() // Timeline pauses here
  .to(".box2", { x: 200 });

// Resume with: tl.resume()
```

## üìñ Summary

**Timeline = Animation Orchestra Conductor**

- **Sequential**: Default behavior (one after another)
- **Overlapping**: Use `"-=1"` (start before previous ends)
- **Simultaneous**: Use `"<"` (start at same time)
- **Labels**: Named checkpoints for complex timing
- **Control**: `.play()`, `.pause()`, `.reverse()`, `.restart()`
- **Speed**: `.timeScale(2)` for 2x speed

**Key Takeaway**: Timelines make complex animation sequences simple, maintainable, and controllable!
